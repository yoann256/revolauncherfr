# Animations de chaînes
La mécanique clé de **Revo Launcher** est la possibilité de créer des **animations de chaînes**, à la fois pour l’icône (animation dans le menu principal) et pour la bannière (animation lorsque vous ouvrez une chaîne).  
Elles peuvent sembler très confuses à utiliser au début. Cependant, vous pouvez tout apprendre à leur sujet ici afin de créer vos propres animations !

# Éditeur d’animations de chaînes
Vous pouvez accéder à l’éditeur d’animations via le **bouton Revo**. À l’intérieur, vous pouvez choisir de modifier les animations d’icône ou de bannière existantes, ou même d’en créer de nouvelles !  
N’hésitez pas à consulter celles déjà créées afin d’en apprendre davantage à leur sujet.

Lors de la modification d’une animation, vous pouvez la prévisualiser en cliquant sur **« Prévisualiser »** puis sur le bouton **« Jouer »**.  

:::tip
Vous pouvez également prévisualiser n’importe quelle animation de la liste en maintenant appuyé son titre.
:::

# Types d’animations de chaîne
## Animations d’icônes
Les animations d’icônes sont plus petites que les animations de bannière, donc il n’y a pas de problème à utiliser des images de moindre qualité.

Lors de l’édition d’une animation d’icône, vous verrez une case **« Durée de la Boucle : »**. Cela est dû au fait que toutes les animations d’icônes bouclent, donc assurez-vous que votre animation revienne à un état initial correct à la fin de la boucle !

**Résolution :** 930x700 px

## Animations de bannières
Les animations de bannière occupent généralement tout l’écran du téléphone et jouent un son lorsqu’elles sont lancées.  

**Résolution :** 1080x1650 px

# Images
Chaque animation de chaîne est composée de plusieurs images qui exécutent différentes animations.

Si vous cliquez sur **« Edit images »**, vous trouverez la liste des images dans l’animation.  

:::tip
Si vous maintenez le bouton **« Edit »** d’une image, vous pouvez la cloner ! (animations incluses)
:::

En faisant défiler vers le bas, vous trouverez **« Ajouter Image »**, **« Ajouter icône d'appli »** et **« Ajouter Vidéo »**.  
Ils se comportent de la même manière, mais **« Icône d'appli »** utilisera comme image l’icône de l’application liée à votre chaîne (vous pouvez ainsi créer des animations génériques qui auront un aspect différent selon chaque application liée, comme celles par défaut !).  
Quant aux vidéos, vous pouvez utiliser des fichiers **.mp4** dans vos animations. Cependant, gardez à l’esprit qu’elles nécessitent quelques images pour le tampon, donc elles donnent un meilleur rendu dans les icônes plutôt que dans les bannières (mais n’hésitez pas à essayer !).

### Champs disponibles

- **Image**
  - L’image utilisée pour cette image animée. Le format attendu est **.png**, **.jpg** ou **.gif** (vous pouvez donc utiliser des images animées !)
- **Position, échelle et rotation initiales**
  - Définit ces valeurs initiales lorsque l’image apparaît dans l’animation (c’est-à-dire lorsque le temps de début est atteint).
  - **Remarque :** la caméra de l’application est **orthographique**, ce qui signifie que les rotations XY ne changeront pas beaucoup l’apparence (l’image semblera écrasée plutôt que tournée). La rotation Z fonctionnera comme prévu.
- **Calque (Layer)**
  - Définit dans quel ordre les images animées seront dessinées. Les calques inférieurs envoient les éléments à l’arrière-plan, tandis que les calques supérieurs les placent au premier plan.
  - Cela ne modifie ni l’échelle ni la position des images, seulement l’ordre dans lequel elles sont dessinées.
  - Les valeurs négatives sont autorisées.
- **Couleur initiale**
  - La couleur avec laquelle l’image sera teintée. Cela fonctionne comme un filtre — **blanc = image inchangée.**
- **Temps de début et de fin**
  - Ces deux valeurs importantes spécifient à quel moment dans la chronologie l’image apparaîtra.
    - **Start time :** 0 signifie qu’elle apparaît dès le début de l’animation ; des valeurs plus élevées la font apparaître plus tard.
    - **End time :** -1 signifie que l’image ne disparaît jamais une fois apparue. Sinon, indiquez une valeur (supérieure au temps de début) pour qu’elle disparaisse.

Enfin, vous avez le bouton **« Modifider l'anim. »**, qui vous permet d’ajouter diverses animations aux images.

# Animations d’images
Les animations d’images peuvent être créées via le bouton **« Modifier l'anim »**.  
Cliquer sur le nom d’une animation affichera une fenêtre contextuelle expliquant ce que fait chaque animation.

Toutes les animations ont leurs propres paramètres, mais les plus courants sont les suivants :

- **Début**
  - Indique à quel moment de la chronologie cette animation spécifique doit commencer (par exemple, quand l’image doit commencer à bouger).
- **Fin**
  - Indique à quel moment de la chronologie cette animation doit se terminer (par exemple, quand le mouvement doit s’arrêter).
- **En boucle ?**
  - C’est un booléen qui vous permet de choisir si cette animation doit boucler ou non.
  - Cela signifie qu’une fois l’animation terminée, elle recommence avec les mêmes paramètres, indéfiniment.
  - **Remarque :** cela ne réinitialisera pas l’image à son état précédent. Pour créer un mouvement aller-retour, créez deux animations distinctes.
- **Délai de la boucle**
  - Si la case ci-dessus est cochée, cela indique combien de temps l’animation doit attendre avant de recommencer (elle attend un peu après la fin avant de redémarrer).
  - Cela est généralement utilisé pour **enchaîner** des animations.  
    Exemple : si une animation doit boucler un mouvement vers l’avant de 1s à 3s, puis en arrière de 3s à 5s, vous voudrez ajouter un délai de 2s à chacune.
- **Ease**
  - Cette option n’apparaît pas dans toutes les animations.  
    Elle permet de régler la vitesse de changement des valeurs au cours de l’animation.
  - Vous pouvez voir facilement toutes les interpolations ici : [https://easings.net/](https://easings.net/)

Enfin, chaque animation contiendra ses propres paramètres (mouvement, rotation, ombre, etc.).

:::tip
Si vous ne savez pas ce que signifient certains paramètres, essayez ! Vous pouvez toujours prévisualiser avec **« Preview » → « Play »**.
:::

# Animations de trajectoire
Une animation plus unique est l’animation de **trajectoire**.  
Elle vous permet de définir un certain nombre de **points de passage** le long desquels une image doit se déplacer, pour un mouvement plus fluide ou courbé.

Les types de trajectoire **Linear** et **CatmullRom** suffiront dans 90 % des cas !  
Cependant, il existe un type spécial appelé **« Cubic Bézier »**.  
C’est une méthode avancée de création de courbes, et un standard des animations sur Internet. Vous pouvez trouver beaucoup d’informations à ce sujet en ligne.

Si vous utilisez une trajectoire **Cubic Bézier**, gardez à l’esprit qu’elle attend que vous ajoutiez les points de passage par **groupes de 3**, comme illustré ici :

# Où les animations sont-elles enregistrées ?
Les chaînes et les animations sont enregistrées dans :

- **Android :**  
  `Internal Storage\Android\data\com.karuzohikari.wiiphone\files\<Channels or Animations>`
- **Windows :**  
  `C:\Users\<Your user>\AppData\LocalLow\KaruzoHikari\Revo Launcher\<Channels or Animations>`

Vous pouvez sauvegarder vos fichiers de cette manière, et même envoyer des animations et des dispositions de menu principal à vos amis !  
(Elles seront chargées au démarrage de l’application si vous les placez dans les bons dossiers.)
